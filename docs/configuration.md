# Configuration and secrets

Arclya loads configuration from **environment variables**. For local development, copy `.env.example` to `.env` at the repository root; the server loads it automatically on startup.

**Never commit `.env`** — it is gitignored. On Render, set variables in the service **Environment** tab (or via `render.yaml`).

## Quick setup (local)

```bash
cp .env.example .env
# Edit .env with your keys
python -m arclya2a.server.app
```

Minimum for local dev:

| Variable | Required? | Notes |
|----------|-----------|-------|
| `XAI_API_KEY` | For live LLM | Tests and `--mock` demos work without it |
| `ARCLYA_API_KEY` | Production only | Leave unset locally to disable API auth |
| `ARCLYA_OPERATOR_KEY` | For graduation | Needed to run `graduate_partner.py` or `POST /partners/graduate` |

## Configuration module

All settings are centralized in `src/arclya2a/settings.py`:

```python
from arclya2a.settings import get_settings

settings = get_settings()
settings.arclya_api_key
settings.crypto.wallet_address
```

`get_settings()` reads the current environment on each call and loads `.env` once when present.

## Render (production)

Render injects environment variables at runtime. Typical production setup:

| Variable | How to set |
|----------|------------|
| `ARCLYA_API_KEY` | Auto-generated by `render.yaml` blueprint, or set manually |
| `XAI_API_KEY` | Set manually in Render dashboard |
| `ARCLYA_OPERATOR_KEY` | Generate a strong secret (32+ chars); never share with partners |
| `RENDER_EXTERNAL_URL` | Set automatically by Render |
| `PORT` | Set automatically by Render |

Crypto payment variables (when ready to accept USDC on multiple chains):

| Variable | Example |
|----------|---------|
| `ARCLYA_CRYPTO_ENABLED` | `1` |
| `ARCLYA_CRYPTO_NETWORKS` | `base,ethereum,solana,bnb` |
| `ARCLYA_CRYPTO_WALLET_BASE` | `0x…` (Base USDC) |
| `ARCLYA_CRYPTO_WALLET_ETHEREUM` | `0x…` (Ethereum USDC) |
| `ARCLYA_CRYPTO_WALLET_SOLANA` | `So1…` (Solana USDC) |
| `ARCLYA_CRYPTO_WALLET_BNB` | `0x…` (BNB Smart Chain USDC) |
| `ARCLYA_CRYPTO_NETWORK` | Default chain for intents (`base`) |
| `ARCLYA_CRYPTO_TOKEN` | `USDC` |

Aliases accepted in `ARCLYA_CRYPTO_NETWORKS`: `eth` → ethereum, `bsc` → bnb.

## Crypto payments — what you need

**You do not need a wallet private key in `.env`.** Only configure a **public receive address** where customers send USDC.

1. Create or use an existing wallet (Coinbase Wallet, MetaMask, Phantom, etc.).
2. Copy your **public address** for the network you choose (e.g. Base mainnet for low-fee USDC).
3. Set per-network receive addresses (`ARCLYA_CRYPTO_WALLET_BASE`, `_ETHEREUM`, `_SOLANA`, `_BNB`).
4. Set `ARCLYA_CRYPTO_NETWORKS=base,ethereum,solana,bnb` and `ARCLYA_CRYPTO_ENABLED=1`.

Agents submit `tx_hash` proof via `POST /payments/crypto/{payment_id}/submit`. **Automated on-chain verification is not implemented yet** — operators confirm payments manually after checking the transaction on a block explorer (`scripts/confirm_crypto_payment.py` or `POST /payments/crypto/{payment_id}/confirm`). Payment records live in `data/payments/crypto_payments.jsonl`.

**Security rules:**

- Never put seed phrases, private keys, or keystore JSON in `.env` or the repo.
- Use a dedicated receive address for Arclya sales; rotate if compromised.
- Prefer Base + USDC for Phase 1 (low fees, widely supported).

## Variable reference

See `.env.example` for the full list with defaults. Grouped categories:

- **Core:** `XAI_API_KEY`, `ARCLYA_API_KEY`, `ARCLYA_OPERATOR_KEY`
- **Server:** `ARCLYA_PUBLIC_URL`, `PORT`, `ARCLYA_RATE_LIMIT_PER_MINUTE`, `ARCLYA_JSON_LOGS`
- **Sandbox:** `ARCLYA_SANDBOX_*`
- **Tools:** `ARCLYA_TOOL_DRY_RUN`, retry settings
- **Learning:** scheduler and auto-apply settings
- **Graduation:** `ARCLYA_GRADUATION_WEBHOOK_URL`
- **Crypto:** `ARCLYA_CRYPTO_*`

## Rotating secrets

1. Generate a new value locally.
2. Update Render environment (or `.env` for local).
3. Redeploy or restart the service.
4. Notify partners if `ARCLYA_API_KEY` changed.
